Built with

The tech behind the jars.

My Jars is built entirely with Apple's native frameworks — no third-party dependencies, no external services, no backend infrastructure of any kind.

SwiftUI

Declarative UI framework

The entire app is built in SwiftUI — Apple's modern framework for building native iOS interfaces with clean, readable code.

UserDefaults

Local persistence

All data is stored on-device using UserDefaults with JSON encoding. No database, no server, no network calls of any kind.

SF Symbols

Apple's icon system

All icons use Apple's SF Symbols — 35 options for jar icons, free to use in App Store apps, no attribution required.

Combine

Reactive data layer

The ObservableObject store uses @Published to keep the UI in sync with data automatically across all views.

StoreKit 2

Review prompts

SKStoreReviewController prompts engaged users for a rating at natural moments — after adding their 3rd, 10th and 30th item.

About

Built by one person, for everyone.

My Jars started as a simple idea — a calm, private place to keep the things you want to remember without the overhead of a notes app or the pressure of a to-do list.

It was built entirely in SwiftUI as a solo hobbyist project. The whole experience — from the warm colour palette to the one-tap item entry — was designed to feel personal and unhurried.

Version 1.0 is deliberately simple. Future updates will add features based on genuine user feedback. If you have a suggestion, I want to hear it.

Privacy Policy — My Jars

Last updated: March 2026

My Jars does not collect, store, transmit or share any personal data. The app has no analytics, no advertising framework, no third-party SDKs, and no backend server of any kind.

Data storage. All data you enter — jar names, items, links — is stored exclusively on your device using iOS UserDefaults. This data never leaves your device and is not accessible to the developer or any third party.

No account required. My Jars does not require you to create an account, provide an email address, or share any personal information to use the app.

No tracking. My Jars does not track your behaviour, location, usage patterns or any other information. There is no analytics code of any kind in the app.

App Store reviews. If you choose to leave a review on the App Store, that review is governed by Apple's own privacy policy, not ours.

Contact. If you contact us by email for support, your email address is used solely to reply to your message and is not stored, shared or used for any other purpose.

Changes. If this policy ever changes, the updated version will be posted at this URL with a new date. Given the nature of this app, significant changes are unlikely.
